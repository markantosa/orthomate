# 🔧 Pumped Up Kicks V2

## Hardware Design Reference (FINAL)

### **Revision V6.0 — ENN hardwired, OLED GPIO6/7, VBAT GPIO15, new J9 connector layout**

---

# 1. System Overview

Battery-powered adaptive orthopedic insole controller.

### Insole:

* 3× micro linear bipolar stepper actuators
* 4× FSR pressure sensors

### Controller:

* ESP32-C6 SuperMini (header-mounted)
* 3× TMC2209 (STEP/DIR mode, header-mounted)
* FSR analog front-end
* TP4056 Type-C charging module
* XL6009 boost converter → 5V system rail
* **0.91" OLED display (I2C)**

---

# 10. Connector Definition

## J_MAIN (20-pin)

**Footprint:** `Connector_Generic:Conn_02x10_Odd/Even`
(Row A = odd pins 1/3/5…19, Row B = even pins 2/4/6…20)

### Physical layout

| Row | Pin A (odd) | Signal A    | Signal B     | Pin B (even) |
| --- | ----------- | ----------- | ------------ | ------------ |
| 1   | 1           | ACT1_OA2    | GND          | 2            |
| 2   | 3           | ACT1_OA1    | GND          | 4            |
| 3   | 5           | ACT1_OB1    | 3V3          | 6            |
| 4   | 7           | ACT1_OB2    | CONN_DETECT  | 8            |
| 5   | 9           | ACT2_OA2    | FSR4         | 10           |
| 6   | 11          | ACT2_OA1    | FSR3         | 12           |
| 7   | 13          | ACT2_OB1    | FSR2         | 14           |
| 8   | 15          | ACT2_OB2    | FSR1         | 16           |
| 9   | 17          | ACT3_OA2    | ACT3_OB2     | 18           |
| 10  | 19          | ACT3_OA1    | ACT3_OB1     | 20           |

### Per-pin reference

### Loopback detect circuit

```
Controller PCB:         Insole PCB:
pin 6 (3V3) ──────────── 10k ──── pin 8 (CONN_DETECT)
                                        │
                                   GPIO15 (INPUT_PULLDOWN, ~45k internal)
                                        │
                                       GND
```

* Unplugged → GPIO5 pulled LOW by internal pulldown → CONN_DETECT = 0
* Plugged in → 3V3 through insole 10k + internal ~45k divider → **V_GPIO5 ≈ 2.7V** → CONN_DETECT = 1
* No external pulldown resistor needed on controller PCB
* Insole only needs one 0402 10k between pin 1 and pin 2

### Per-pin reference

| Pin | Signal       | Notes                                                      |
| --- | ------------ | ---------------------------------------------------------- |
| 1   | ACT1_OA2     | TMC2209 OA2 → ACT1 coil A−                                |

# 11. PCB Zoning

## Zone A — Power

* LiPo
* TP4056
* XL6009
* bulk caps

## Zone B — Stepper

* TMC2209
* motor outputs

## Zone C — Logic + Analog

* ESP32
* FSR circuits
* OLED

> Keep Zone C far from boost + motors 

---

# 12. Ground Strategy

**Trunk + Branch**

* Battery GND = origin
* Separate:

  * Power return
  * Logic return
  * Analog return

> NEVER mix motor return with analog ground 

---

# 13. Routing Priorities

1. Power
2. Motor outputs
3. STEP/DIR
4. Analog
5. I2C

---

## Trace Widths

| Type    | Width       |
| ------- | ----------- |
| Power   | 1.5–2.0 mm  |
| Motor   | 0.8–1.2 mm  |
| Digital | 0.25–0.4 mm |
| Analog  | 0.25–0.4 mm |

---

# 14. Boost Layout Rules

* Keep far from analog
* Minimize loop area
* Thick traces
* Treat as **noise source**

---

# 15. Firmware Interaction Rules

## Motion

* Do NOT start all motors at once
* Delay 50–100 ms

## ADC

* Sample when motors idle OR median filter

## OLED

* 5–10 Hz update
* Avoid switching spikes

---

# 16. Bring-Up Sequence

1. Power only
2. Validate ADC + 3.3V
3. Test OLED
4. Test 1 driver
5. Scale

---

# 17. Risks & Mitigation

## R1 — FSR Noise

* RC filter
* zoning

## R2 — Power instability

* bulk caps

## R3 — Charging conflict

* firmware lockout

## R4 — Boot issues (NEW)

* avoid pulling GPIO4 during boot
